# 简要描述示例 - 输入格式参考

本文档提供几个简要描述的示例，展示用户应该如何描述功能。

---

## 示例1：每日签到（简单功能）

```markdown
# 每日签到

## 核心玩法
玩家每天登录后可以在签到界面点击签到按钮，领取当日的签到奖励。
连续签到天数越多，奖励越丰富。

## 主要规则
- 每日可签到1次，每天00:00刷新
- 连续签到7天为一个周期，完成后重新开始
- 如果中断签到，连续天数清零，重新从第1天开始计算

## 关键数值
- 第1天奖励：金币100
- 第7天奖励：金币500 + 稀有宝箱×1
- 每天奖励递增

## 特殊说明
- 需要支持补签功能（消耗钻石）
- VIP玩家可以双倍领奖
```

## 示例2：装备强化（中等复杂度）

```markdown
# 装备强化系统

## 核心玩法
玩家可以消耗强化石和金币来强化装备，提升装备的属性。
强化有成功率，失败时不会损失装备，但会浪费强化石。

## 主要规则
- 每件装备可以强化到+15
- 强化等级越高，成功率越低
- 强化成功后，装备属性按百分比提升
- 失败后强化等级不变，材料消耗

## 关键数值
- +1到+5：成功率100%
- +6到+10：成功率递减（90%、80%、70%、60%、50%）
- +11到+15：成功率递减（40%、30%、20%、10%、5%）
- 每次强化消耗：强化石×1 + 金币（等级×1000）

## 特殊说明
- 强化+10、+15时需要播放特效
- 可以使用保护符防止失败（消耗钻石）
- 强化等级会显示在装备图标上
```

## 示例3：公会战（复杂功能）

```markdown
# 公会战系统

## 核心玩法
每周末开启公会战活动，不同公会之间进行对战，争夺排名和奖励。
公会成员可以攻击敌方公会的据点，摧毁所有据点的公会获胜。

## 主要规则
1. **开启时间**：每周六、周日 20:00-21:00
2. **参与条件**：公会等级≥3，公会成员≥10人
3. **匹配规则**：根据公会战力匹配相近的对手
4. **战斗规则**：
   - 每个公会有5个据点，每个据点有耐久度
   - 公会成员可以攻击敌方据点，每次攻击消耗1点行动力
   - 摧毁所有敌方据点后获胜
   - 时间结束时剩余据点多的公会获胜
5. **奖励规则**：根据公会排名发放奖励，个人根据贡献获得奖励

## 关键数值
- 每人每天行动力：5点
- 据点初始耐久度：10000
- 每次攻击伤害：与玩家战力相关
- 排名奖励：第1名（钻石500）、第2-3名（钻石300）、第4-10名（钻石100）

## 特殊说明
- 需要实时战况广播
- 需要公会聊天频道
- 攻击据点时显示战斗动画
- 公会会长可以调整防守阵型
```

## 示例4：限时活动（运营活动案例）

```markdown
# 充值返利活动

## 核心玩法
活动期间，玩家充值可以获得额外的返利奖励。
充值金额越高，返利比例越高，并可以领取档位奖励。

## 主要规则
- 活动持续7天
- 玩家充值任意金额，都可以获得基础返利（10%）
- 累计充值达到不同档位，返利比例提升，并解锁档位奖励
- 返利奖励以绑定钻石形式发放
- 档位奖励需要手动领取

## 关键数值
| 充值档位 | 返利比例 | 档位奖励 |
|----------|----------|----------|
| 100元 | 10% | 强化石×10 |
| 300元 | 15% | 强化石×30 + 稀有宝箱×1 |
| 500元 | 20% | 强化石×50 + 史诗宝箱×1 |
| 1000元 | 25% | 强化石×100 + 传说宝箱×1 + 限定坐骑 |

## 特殊说明
- 活动结束后，未领取的档位奖励发送邮件
- 活动期间充值多次，累计计算
- 需要在主界面显示活动入口和倒计时
```

## 推荐的描述格式

### 最小版本（极简单功能）
```markdown
# 功能名称

## 核心玩法
[2-3句话描述玩法]

## 主要规则
- 规则1
- 规则2
```

### 标准版本（适合大多数功能）
```markdown
# 功能名称

## 核心玩法
[1-3段话描述核心玩法流程和玩家操作]

## 主要规则
- 规则1
- 规则2
- ...

## 关键数值
- 参数1: 值
- 参数2: 值
```

### 完整版本（复杂功能）
```markdown
# 功能名称

## 核心玩法
[详细描述]

## 主要规则
1. 规则类别1
   - 细节1
   - 细节2
2. 规则类别2
   - 细节1

## 关键数值
[表格或列表]

## 特殊说明
[特殊要求、注意事项]
```

## 编写建议

### ✅ 应该包含
- 玩家的操作流程
- 核心的游戏规则
- 重要的数值（至少关键节点）
- 特殊的要求或限制

### ❌ 不需要包含
- UI布局和美术需求（这不在文档范围内）
- 详尽的所有数值（AI会帮你扩展）
- 完整的配置表结构（AI会设计）
- 所有的特殊情况（AI会推导）

### 💡 描述技巧
1. **核心玩法**用白话描述，就像给朋友介绍这个功能
2. **主要规则**列出最关键的3-5条，不用面面俱到
3. **关键数值**提供代表性的数值，AI能推导其他值
4. **不确定的地方**可以用"大约"、"建议"等词，AI会帮你具体化

记住：简要描述是**框架**，AI会帮你**填充细节**。你只需提供核心思路，无需事无巨细。
